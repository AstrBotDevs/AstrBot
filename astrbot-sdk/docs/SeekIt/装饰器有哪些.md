# 触发器装饰器（5个）
## 这些装饰器决定何时执行你的方法：

@on_command - 命令触发器，支持别名和描述
@on_message - 消息触发器，支持关键词或正则匹配
@on_event - 事件触发器，响应系统事件
@on_schedule - 定时任务触发器，支持 cron 表达式或固定间隔
@conversation_command - 带会话生命周期的命令触发器，内置超时和冲突处理
详见 
decorators.pyL452-L660

## 权限与过滤装饰器（6个）
这些装饰器限定谁能触发，在哪触发：

@require_admin / @admin_only - 管理员权限标记（两者等价）
@require_permission - 通用角色权限标记（"member" 或 "admin"）
@platforms - 限定特定平台（如 @platforms("qq", "discord")）
@message_types - 消息类型过滤（如 @message_types("group")）
@group_only - 仅群聊触发
@private_only - 仅私聊触发
详见 
decorators.pyL833-L915

## 限流装饰器（2个）
防止滥用的关键机制：

@rate_limit - 滑动窗口限流（如 @rate_limit(5, 60.0) = 60秒内最多5次）
@cooldown - 冷却时间（如 @cooldown(30.0) = 每30秒最多触发1次）
两者都支持作用域（session/user/group/global）和行为（hint/silent/error）配置。详见 
decorators.pyL929-L990

## 能力导出装饰器（7个）
这些装饰器让插件暴露能力给外部或运行时：

@provide_capability - 声明对外暴露的能力，支持流式和取消
@register_llm_tool - 注册 LLM 工具，供语言模型调用
@register_agent - 注册 Agent 类
@http_api - 暴露 HTTP API 端点
@validate_config - 配置验证钩子（支持 Pydantic 模型或 JSON Schema）
@on_provider_change - 监听 LLM Provider 变更
@background_task - 后台任务标记，支持自动重启
@mcp_server - 声明 MCP 服务器实例
@register_skill - 注册技能（Skill）
详见 
decorators.pyL660-L832

重要原则：装饰器顺序不影响元数据收集。它们只是在方法上附加元数据，由 Star.__init_subclass__ 在类定义时自动收集。修饰装饰器（如 @require_admin）必须与触发器装饰器（如 @on_command）配对使用。
