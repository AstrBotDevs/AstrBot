# 消息事件处理

"消息事件处理"就是：用户消息 → 装饰器匹配 → 你的代码执行 → 发送响应

消息事件（MessageEvent）：封装了一条用户消息的所有信息（文本、发送者、平台、群聊/私聊等）
处理（Handler）：你用装饰器（如 @on_command、@on_message）标记的函数，它们会在特定条件下被触发


## 具体来说
当用户在 QQ 群发送 /hello，框架做了这些事：

### 1. 平台适配器把原始消息转换成 MessageEvent
```python
event = MessageEvent(
    text="/hello",
    user_id="12345",
    group_id="67890",
    platform="qq",
    # ... 其他字段
)
```

### 2. 运行时找到匹配的 handler（通过装饰器声明的触发器）
@on_command("hello")  # <-- 这个装饰器告诉运行时：当收到 /hello 命令时调用我
async def hello(self, event: MessageEvent, ctx: Context):
    await event.reply("Hello!")  # <-- 这就是"处理"
 
### 3. 运行时执行你的函数，注入 event 和 ctx 参数
### 4. 你调用 event.reply() 发送响应


## MessageEvent 是什么?
它是一个数据容器，包含了：

消息内容：event.text 是纯文本，event.get_messages() 是富媒体组件（图片、@、文件等）
身份信息：event.user_id、event.group_id、event.is_admin()
回复能力：event.reply()、event.reply_image()、event.reply_chain()
参见 
src\astrbot_sdk\events.py L108-L224


## 装饰器系统如何工作
装饰器在方法上附加元数据（HandlerMeta），运行时启动时扫描这些元数据，构建一张"触发器 → 处理函数"的映射表。当消息到来时，HandlerDispatcher 根据触发器匹配规则决定调用哪个函数。

参见 
src\astrbot_sdk\decorators.py L1-L150

完整流程图
用户消息 (QQ/Discord/...)
    ↓
平台适配器（转换为 MessageEvent）
    ↓
CapabilityDispatcher（识别目标 handler）
    ↓
HandlerDispatcher（评估过滤器、权限、限流）
    ↓
参数注入（event, ctx, 自定义参数）
    ↓
你的 handler 函数执行
    ↓
返回结果或调用 event.reply()
    ↓
发送响应到平台

## 为什么叫"事件"而不是"消息"
因为不只是文字消息。框架也支持：

定时事件（@on_schedule）
平台事件（@on_event，比如群成员增加、llm回复完毕和好友申请等）
会话事件（用户在等待输入时的回复）
所以统一用 MessageEvent 这个名字，虽然它主要承载消息数据。

## 实际例子
```python
from astrbot_sdk import Star, MessageEvent, Context
from astrbot_sdk.decorators import on_command, on_message, require_admin
 
class MyPlugin(Star):
    @on_command("echo")  # 触发器：/echo 命令
    async def echo(self, event: MessageEvent, ctx: Context):
        # event.text 已经去掉了 /echo 前缀，只剩下参数
        await event.reply(f"你说：{event.text}")
    
    @on_message(keywords=["帮助", "help"])  # 触发器：消息包含关键词
    async def help(self, event: MessageEvent):
        await event.reply("可用命令：/echo, /status")
    
    @on_command("ban")
    @require_admin  # 权限过滤器：只有管理员能触发
    async def ban_user(self, event: MessageEvent, ctx: Context):
        await event.reply("用户已被封禁")
```
当用户发送 /echo 测试 时：

运行时匹配到 echo 方法的 @on_command("echo") 触发器
创建 MessageEvent 实例，text 字段是 "测试"（命令前缀被移除）
注入 event 和 ctx 参数，调用 echo 方法
你的代码调用 event.reply()，响应发送回用户
